# User Story

## Связывание с модификаторами

**Роль:** администратор.

**Цель:** привязать к услуге один или несколько модификаторов, доступных пользователю при оформлении заказа.

**Предусловие:** администратор открыл форму создания или редактирования услуги и перешел в раздел привязки модификаторов.

**Основной сценарий:**

1. Администратор выбирает модификатор из списка доступных модификаторов.
2. Система отображает значения выбранного модификатора.
3. Администратор выбирает значения модификатора, доступные для данной услуги.
	1. Alt. `Выбор всех значений`
4. Администратор указывает, является ли выбор модификатора обязательным.
5. Администратор указывает добавочную стоимость для каждого выбранного значения модификатора.
6. Система добавляет модификатор к услуге.
7. Администратор завершает настройку.
	1. Alt. `Добавление очередного модификатора`
	2. Alt. `Модификатор не выбран`

**Выбор всех значений:**
3. Администратор выбирает действие «все значения», и система добавляет к услуге все значения выбранного модификатора.
4. Возврат к пункту 4 **основного сценария**.

**Добавление очередного модификатора**
7. Возврат к пункту 1

**Модификатор не выбран:**
7. Администратор пытается завершить настройку без выбранного модификатора.
8. Система сообщает, что услуге должен быть назначен хотя бы один модификатор.
9. Возврат к пункту 1 **основного сценария**.

**Результат:** услуга содержит список привязанных модификаторов, доступных значений, признаков обязательности и добавочных стоимостей.


## Привязка категории

**Роль:** администратор.

**Цель:** назначить услуге одну или несколько категорий услуги.

**Предусловие:** администратор открыл форму создания или редактирования услуги и перешел в раздел привязки категорий.

**Основной сценарий:**

1. Администратор запрашивает список доступных категорий услуги.
2. Система отображает доступные категории услуги.
3. Администратор выбирает одну или несколько категорий услуги.
	1. Alt. `Использование категории «Другое»`
4. Система добавляет выбранные категории к услуге.
5. Администратор завершает настройку.
	1. Alt. `Категория не выбрана`

**Использование категории «Другое»:**
3. Администратор выбирает категорию «Другое», если услуге невозможно назначить семантическую категорию.
4. Возврат к пункту 4 **основного сценария**.

**Категория не выбрана:**
5. Система сообщает, что услуге должна быть назначена хотя бы одна категория.
6. Возврат к пункту 3 **основного сценария**.

**Результат:** услуга содержит список назначенных категорий услуги.

# Сущности

# Услуга

Сущность услуги для любых типов услуг.

Статусная модель.

1. Активна
2. Удалена

## Категория услуги

Услуге могут назначаться множество категориий, например, _платья_, _нижняя одежда_, _футболки_, _украшения_ и т.п. 

Каждой услуге **обязательно** должна быть присвоена **хотя бы одна** категория. Если услуге невозможно присвоить какую-либо семантическую категорию, то может использоваться категория **"Другое"**.

## Результат выполнения услуги

В результате выполнения услуги **ВСЕГДА** в системе должна фиксироваться некая сущность.
Для этой сущности **должна** быть указана конечная стоимость. 

# Механики

## Связывание с модификаторами

Связывание с услугой происходит при создании услуги. Для каждой услуги можно выбрать множество существующих модификаторов и множество значений каждого модификатора, которые могут быть выбраны для этой услуги.

При связывании услуги для каждого значения можно указать **обязательную** добавочную стоимость.

Таким образом, она будет учитываться в финальной стоимости услуги при выборе модификатора.
## Конечная стоимость

Конечная стоимость рассчитывается по следующей формуле:

$$
\begin{aligned}
P = wp + mp_{i} \\ \\
\text{where: } \\
& wp      && \text{базовая стоимость услуги} \\
& mp_{i} = 0 && \text{сумма стоимости выбранных значений модификаторов} \\
\end{aligned}
$$
