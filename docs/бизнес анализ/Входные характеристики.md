Документ содержит информацию о входных характеристиках, их связи с пользователем и УСП.
# User Story

## Создание входной характеристики

**Роль:** администратор.

**Цель:** создать входную характеристику, которую можно использовать при настройке УСП.

**Предусловие:** администратор открыл форму создания входной характеристики.

**Основной сценарий:**

1. Администратор указывает название входной характеристики.
2. Администратор указывает описание входной характеристики при необходимости.
3. Администратор выбирает тип входной характеристики: `видео-файл`, `фото-файл`, `текст`, `единица измерения`, `числовое значение` или `да\нет`.
	1. Alt. `Тип не выбран`
4. Администратор сохраняет входную характеристику.
5. Система создает входную характеристику.

**Тип не выбран:**
3. Система сообщает, что тип входной характеристики должен быть выбран.
4. Возврат к пункту 3 **основного сценария**.

**Результат:** входная характеристика создана и доступна для привязки к УСП.

## Создание шаблона входных характеристик

**Роль:** администратор.

**Цель:** создать шаблон входных характеристик для повторного использования при настройке УСП.

**Предусловие:** администратор открыл форму создания шаблона входных характеристик.

**Основной сценарий:**

1. Администратор указывает название шаблона входных характеристик.
2. Администратор указывает описание шаблона при необходимости.
3. Администратор выбирает входные характеристики для включения в шаблон.
	1. Alt. `Характеристики не выбраны`
4. Администратор сохраняет шаблон входных характеристик.
5. Система создает шаблон входных характеристик.

**Характеристики не выбраны:**
3. Система сообщает, что шаблон должен содержать хотя бы одну входную характеристику.
4. Возврат к пункту 3 **основного сценария**.

**Результат:** шаблон входных характеристик создан и доступен для использования при настройке УСП.

# Сущности

## Входная характеристика

Входная характеристика - это единица информации, которая необходима для выполнения услуги. 

Единица информации - это обобщение над различными реальными данными. Под этим может подразумеваться видео-файл, фото-файл, текст, единица измерения, числовое значение, да\нет.

Некоторые характеристики могут быть опциональными, однако опциональность определяется при привязке характеристики к услуге.
## Шаблон входных характеристик

Сущность, которая позволяет объединить множество характеристик в коллекцию.

# Структуры
**Входная характеристика**

| Название поля | Описание поля                                                              |
| ------------- | -------------------------------------------------------------------------- |
| Название      | Текстовое название                                                         |
| Описание?     | Текстовое описание                                                         |
| Тип           | видео-файл, фото-файл, текст, единица измерения, числовое значение, да\нет |

**Шаблон входных характеристик**

| Название поля                | Описание поля                                |
| ---------------------------- | -------------------------------------------- |
| Название                     | Текстовое название                           |
| Описание?                    | Текстовое описание                           |
| Список входных характеристик | Список идентификаторов входных характеристик |
