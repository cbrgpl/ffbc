# User Story

## Добавление\изменение УСП

**Роль:** администратор.

**Цель:** создать или изменить услугу создания предмета, доступную клиенту на платформе.

**Предусловие:** администратор открыл форму создания или редактирования УСП.

**Основной сценарий:**

1. Администратор указывает название УСП.
2. Администратор указывает описание УСП.
3. Администратор указывает базовую стоимость УСП.
4. Администратор добавляет медиа файлы для УСП.
5. Администратор выполняет [[Услуга#Привязка категории|выбор категории]].
6. Администратор выбирает входные характеристики из списка существующих или использует характеристики шаблона полностью либо частично.
	1. `Alt.` Создание входных характеристик
	2. `Alt.` Входные характеристики не выбраны
7. Администратор подтверждает выбранные характеристики.
8. Администратор указывает обязательность каждой входной характеристики.
9. Администратор выполняет [[Услуга#Связывание с модификаторами|выбор модификатора]].
	1. `Alt.` Модификаторы не требуются
10. Администратор сохраняет УСП.
11. Система создает\обновляет УСП.

**Входные характеристики не выбраны:**
6. Система сообщает, что для УСП должна быть выбрана хотя бы одна входная характеристика.
7. Возврат к пункту 6 **основного сценария**.

**Создание входных характеристик**
6. Администратор выполняет [[Входные характеристики#Создание входной характеристики|создание входной характеристики]]
7. Администратор завершает создание входной характеристики.
	1. `Alt.` Создание еще одной характеристики
8. Возврат к пункту 7 **основного сценария**.

**Создание еще одной характеристики:**
9. Администратор инициирует создание очередной входной характеристики.
10. Возврат к пункту 6 **сценария Создание входных характеристик**.

**Модификаторы не требуются:**
9. Возврат к пункту 10 **основного сценария**.

**Результат:** УСП создана или обновлена и содержит название, описание, стоимость, медиа файлы, категории, входные характеристики и, при необходимости, модификаторы.

## Удаление УСП

**Роль:** администратор.

**Цель:** удалить УСП из системы.

**Предусловие:** администратор открыл список УСП.

**Основной сценарий:**

1. Администратор выбирает УСП.
2. Администратор запрашивает удаление УСП.
3. Система запрашивает подтверждение удаления.
4. Администратор указывает, нужно ли удалить связанные входные характеристики.
5. Администратор подтверждает удаление.
	1. `Alt.` Отмена удаления
6. Система удаляет модификаторы, связанные с УСП.
7. Система удаляет входные характеристики, связанные с УСП и не связанные с другой УСП или шаблоном.
8. Система удаляет выбранную УСП.
9. Система исключает УСП из списка активных услуг.

**Отмена удаления:**
5. Администратор отменяет удаление.
6. Система оставляет УСП без изменений.

**Результат:** УСП, связанные модификаторы и выбранные входные характеристики удалены из системы.

## Добавление УСП в корзину

**Роль:** клиент.

**Цель:** добавить УСП в корзину для последующего оформления заказа.

**Предусловие:** клиент находится в карточке УСП.

**Основной сценарий:**

1. Клиент просматривает описание, медиа файлы и стоимость УСП.
2. Система отображает доступные значения модификаторов для УСП.
	1. `Alt.` Модификаторы отсутствуют
3. Клиент выбирает значения модификаторов.
4. Система отображает входные характеристики, необходимые для выполнения УСП.
5. Клиент заполняет обязательные входные характеристики.
	1. `Alt.` Обязательные характеристики не заполнены
6. Клиент заполняет опциональные входные характеристики при необходимости.
7. Система рассчитывает [[Услуга#Конечная стоимость|конечную стоимость]] УСП.
8. Клиент добавляет УСП в корзину.
9. Система добавляет в корзину созданный предмет с выбранными значениями модификаторов, заполненными входными характеристиками и конечной стоимостью.

**Модификаторы отсутствуют:**
2. Система не отображает выбор модификаторов.
3. Возврат к пункту 4 **основного сценария**.

**Обязательные характеристики не заполнены:**
5. Система сообщает, что обязательные входные характеристики должны быть заполнены.
6. Возврат к пункту 5 **основного сценария**.

**Результат:** корзина содержит созданный предмет, сформированный на основе выбранной УСП.

## Редактирование УСП в корзине

**Роль:** клиент.

**Цель:** изменить входные характеристики созданного предмета, добавленного в корзину.

**Предусловие:** корзина содержит созданный предмет, сформированный на основе УСП.

**Основной сценарий:**

1. Клиент открывает корзину.
2. Клиент выбирает созданный предмет.
3. Клиент запрашивает редактирование УСП в корзине.
4. Система отображает заполненные входные характеристики.
5. Клиент изменяет значения входных характеристик.
	1. `Alt.` Обязательные характеристики не заполнены
6. Клиент сохраняет изменения.
7. Система обновляет созданный предмет в корзине.

**Обязательные характеристики не заполнены:**
5. Система сообщает, что обязательные входные характеристики должны быть заполнены.
6. Возврат к пункту 5 **основного сценария**.

**Результат:** созданный предмет в корзине содержит обновленные входные характеристики.

# Сущности

## Услуга создания предмета

Является одним из типов услуг, продаваемых на платформе. Данная услуга подразумевает создание какого-либо материального предмета(одежды, бижутерии и т.д.). 

Для выполнения услуги необходима входная информация: [[Входные характеристики#Входные характеристики|входные характеристики]] и [[Модификатор услуги#Модификатор услуги|модификаторы услуги]] (опционально).

**Результатом выполнения** услуги является [[#Созданный предмет|созданный предмет]].
## Категория услуги

## Входные характеристики

Каждой услуге назначается список входных характеристик. У каждой услуги должна быть хотя бы одна входная характеристика.

_Для УПП, как правило, входные характеристики это мерки._
## Модификаторы услуги

Поддерживаются услугой.
# Созданный предмет

Результатом _УСП_ является _созданный предмет_. 
У созданных предметов существует своя статусная модель:

1. `В работе` (назначается по умолчанию)
2. `Готов`
3. `Проблема` - в случае непредвиденных проблем в создании предмет переходит в этот статус
4. `Отменен` - создание предмета отменено
5. `Выставлен на продажу` - не отображается у клиента. Нужен как внутренний статус в админ панели
# Механики

## Подтверждение услуги

Каждый заказ, который будет содержать _УПП_, должен быть проверен и подтвержден администратором вручную. Для _УПП_ **конечная стоимость** - не фиксированна и может быть изменена администратором при подтверждении заказа.

Только после этого пользователю должна стать доступна оплата.

# Структуры

**УСП**

| Название поля          | Описание поля                                          | Валидации                                                            |
| ---------------------- | ------------------------------------------------------ | -------------------------------------------------------------------- |
| Название               | Текстовое название продукта                            | Обязательно<br>Макс. 250 символов                                    |
| Описание               | Текстовое описание продукта                            | Максимум 1500 символов                                               |
| Категории              | Категории продукта                                     | Минимум 1<br>**"Другое"** не может быть вместе с другими категориями |
| Стоимость              | Базовая стоимость услуги                               | Обязательно                                                          |
| Входные характеристики | Список пар <входная характеристика, обязательность?>   | Минимум 1                                                            |
| Медиа файлы            | Фото\видео файлы, не более 10 шт                       | Минимум 1<br>Форматы: `*formats*`<br>Макс размер: `*max size*`       |
| Модификаторы           | Список модификаторов                                   |                                                                      |
| Значения модификаторов | Список значений модификаторов для каждого модификатора |                                                                      |
