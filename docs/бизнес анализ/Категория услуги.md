# User Story

## Создание категории

**Роль:** администратор.

**Цель:** создать категорию услуги для группировки услуг и определения входных характеристик для УСП.

**Предусловие:** администратор открыл форму создания категории услуги.

**Основной сценарий:**

1. Администратор указывает название категории услуги.
2. Администратор выбирает входные характеристики, связанные с категорией.
	1. Alt. `Входные характеристики не выбраны`
3. Администратор сохраняет категорию услуги.
4. Система создает категорию услуги.

**Входные характеристики не выбраны:**
2. Система сообщает, что для категории услуги должен быть выбран список входных характеристик.
3. Возврат к пункту 2 **основного сценария**.

**Результат:** категория услуги создана и доступна для привязки к услугам.

# Сущности
## Категория услуги

Категория услуги - справочная сущность, которая создается через админ. панель.
Каждой _категории услуги_ может назначаться список входных характеристик, которые должны быть заполнены при оформлении УСП с этой категорией.
